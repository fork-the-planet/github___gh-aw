Security: exclude env vars whose values are job-output expressions (`${{ needs.JOB.outputs.OUTPUT }}`) from the agent container sandbox via AWF `--exclude-env`, in addition to the existing `${{ secrets.* }}` check. This closes the gap where migrating from long-lived PATs stored as secrets to ephemeral tokens generated by an upstream job (a common GitHub App pattern) would silently drop the sandbox exclusion, leaving the token visible inside the AWF container.
